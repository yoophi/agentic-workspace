# Feature Specification: AW Worktree Session Files 하위 디렉터리 조회 수정

**Feature Branch**: `017-fix-filetree-subdirectories`

**Created**: 2026-07-08

**Status**: Draft

**Input**: User description: "$speckit-specify 릴리즈 버전에서 worktreesessionpage 내 files 섹션의 파일트리에서 하위디렉토리의 파일을 조회할 수 없는 이슈 수정"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Files 파일트리에서 하위 디렉터리 파일 열기 (Priority: P1)

Agentic Workbench의 worktree session page를 사용하는 사용자는 릴리즈 버전에서도 Files 섹션의 파일트리에서 하위 디렉터리를 펼치고 그 안의 파일을 선택해 내용을 확인할 수 있다.

**Why this priority**: Files 섹션은 worktree 안의 결과물과 변경 파일을 검토하는 기본 경로이며, 하위 디렉터리 파일 조회 실패는 실제 프로젝트 대부분의 파일 탐색을 막는 핵심 회귀다.

**Independent Test**: 릴리즈 버전에서 두 단계 이상의 하위 디렉터리를 가진 worktree session을 열고, Files 섹션의 파일트리에서 하위 파일을 선택했을 때 해당 파일 내용이 표시되는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 릴리즈 버전의 worktree session page에 하위 디렉터리를 포함한 파일트리가 표시된다, **When** 사용자가 하위 디렉터리를 펼치고 그 안의 파일을 선택한다, **Then** 선택한 파일의 내용이 Files 섹션의 파일 보기 영역에 표시된다.
2. **Given** 사용자가 루트 파일을 선택한 뒤 하위 디렉터리 파일을 선택한다, **When** 선택이 변경된다, **Then** 파일 보기 영역은 이전 파일이 아니라 새로 선택한 하위 파일의 내용을 표시한다.
3. **Given** 사용자가 서로 다른 하위 디렉터리의 파일을 순서대로 선택한다, **When** 각 파일 선택이 완료된다, **Then** 파일트리의 선택 상태와 파일 보기 영역의 표시 파일이 일치한다.

---

### User Story 2 - 기존 루트 파일 조회 동작 유지하기 (Priority: P2)

사용자는 하위 디렉터리 조회 문제가 수정된 뒤에도 루트 파일을 선택하고 조회하는 기존 Files 섹션 동작을 그대로 사용할 수 있다.

**Why this priority**: 회귀 수정은 손상된 하위 파일 조회를 복구하면서 이미 동작하던 루트 파일 조회, 선택 표시, 비어 있는 상태, 오류 표시를 깨뜨리지 않아야 한다.

**Independent Test**: 릴리즈 버전에서 루트 파일과 하위 디렉터리 파일이 함께 있는 worktree session을 열고, 루트 파일 조회와 하위 파일 조회를 번갈아 수행해 두 경로 모두 정상 표시되는지 검증한다.

**Acceptance Scenarios**:

1. **Given** Files 섹션에 루트 파일과 하위 디렉터리 파일이 함께 있다, **When** 사용자가 루트 파일을 선택한다, **Then** 루트 파일 내용은 기존과 같은 방식으로 표시된다.
2. **Given** 사용자가 하위 디렉터리 파일 조회 후 루트 파일로 돌아간다, **When** 루트 파일을 선택한다, **Then** 선택 상태와 파일 보기 내용은 루트 파일로 갱신된다.
3. **Given** 선택한 루트 파일 또는 하위 파일이 읽을 수 없는 상태다, **When** 사용자가 해당 파일을 선택한다, **Then** Files 섹션은 원인을 이해할 수 있는 오류 상태를 표시하고 session page의 다른 영역은 계속 사용할 수 있다.

---

### User Story 3 - 중첩 경로와 특수 경로 파일을 일관되게 처리하기 (Priority: P3)

사용자는 중첩 깊이가 깊거나 이름에 공백, 한글, 특수 문자가 포함된 하위 디렉터리 파일도 릴리즈 버전에서 안정적으로 조회할 수 있다.

**Why this priority**: 실제 worktree에는 다양한 경로와 파일명이 존재하며, 하위 디렉터리 조회 수정은 일반적인 경로뿐 아니라 사용자가 이미 만든 프로젝트 파일명을 폭넓게 지원해야 한다.

**Independent Test**: 여러 단계의 하위 디렉터리, 공백이 포함된 경로, 한글 파일명, 일반적인 기호가 포함된 파일명을 가진 fixture worktree에서 각 파일을 선택해 내용 표시와 선택 상태를 확인한다.

**Acceptance Scenarios**:

1. **Given** 두 단계 이상 중첩된 디렉터리 안에 파일이 있다, **When** 사용자가 파일트리에서 해당 파일을 선택한다, **Then** 파일 내용이 정상적으로 표시된다.
2. **Given** 하위 디렉터리 파일 경로 또는 파일명에 공백이나 한글이 포함되어 있다, **When** 사용자가 해당 파일을 선택한다, **Then** 파일 보기 영역은 올바른 파일을 표시한다.
3. **Given** 같은 파일명이 서로 다른 하위 디렉터리에 존재한다, **When** 사용자가 각각의 파일을 선택한다, **Then** Files 섹션은 선택한 경로의 파일 내용을 구분해 표시한다.

### Edge Cases

- 파일트리에 표시된 하위 파일이 선택 시점에 삭제되었거나 읽을 수 없으면 사용자에게 이해 가능한 오류 상태를 보여주고 현재 session page를 중단하지 않는다.
- 빈 하위 디렉터리는 펼쳐지더라도 파일 보기 영역에 잘못된 이전 파일 내용을 남기지 않는다.
- 같은 이름의 파일이 루트와 하위 디렉터리 또는 서로 다른 하위 디렉터리에 함께 있어도 선택한 전체 경로 기준으로 올바른 파일을 표시한다.
- 공백, 한글, 점, 대시, 밑줄 등 일반적으로 허용되는 경로 문자가 포함된 하위 파일을 선택해도 다른 파일로 잘못 해석하지 않는다.
- 하위 디렉터리를 접거나 다시 펼친 뒤에도 파일 선택과 조회 동작이 일관되게 유지된다.
- 릴리즈 버전과 개발 실행 환경 사이에서 사용자가 체감하는 Files 섹션 파일 조회 동작이 달라지지 않아야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 릴리즈 버전의 worktree session page Files 섹션은 파일트리에 표시된 하위 디렉터리 파일을 사용자가 선택할 수 있어야 한다.
- **FR-002**: 사용자가 하위 디렉터리 파일을 선택하면 Files 섹션은 선택한 파일의 내용을 표시해야 한다.
- **FR-003**: 파일 보기 영역은 사용자가 마지막으로 선택한 파일의 전체 경로와 일치하는 내용을 표시해야 하며, 같은 파일명을 가진 다른 경로의 파일과 혼동하지 않아야 한다.
- **FR-004**: 루트 파일 선택과 조회 동작은 하위 디렉터리 파일 조회 수정 후에도 기존과 동일하게 동작해야 한다.
- **FR-005**: 사용자는 루트 파일, 한 단계 하위 파일, 여러 단계 하위 파일을 순서와 관계없이 선택할 수 있어야 하며, 각 선택마다 표시 내용이 갱신되어야 한다.
- **FR-006**: 파일 또는 디렉터리 이름에 공백, 한글, 점, 대시, 밑줄이 포함되어 있어도 Files 섹션은 올바른 파일을 조회해야 한다.
- **FR-007**: 선택한 하위 파일을 읽을 수 없는 경우 Files 섹션은 사용자가 이해할 수 있는 오류 상태를 표시하고 이전 파일 내용을 새 선택 결과처럼 보여주지 않아야 한다.
- **FR-008**: 하위 디렉터리 파일 조회 실패는 릴리즈 버전에서 재현 가능한 회귀로 다뤄져야 하며, 완료 검증에는 릴리즈 버전 기준의 사용자 흐름 확인이 포함되어야 한다.
- **FR-009**: 파일트리의 펼침, 접힘, 선택 표시 상태는 파일 보기 영역에 표시되는 현재 파일과 일관되어야 한다.
- **FR-010**: 이 수정은 worktree session page의 Files 섹션 범위에 한정하며, agent 실행, prompt 입력, Git 변경 목록, Markdown preview 등 다른 session 영역의 기존 동작을 변경하지 않아야 한다.
- **FR-011**: 기능 검증은 루트 파일, 한 단계 하위 파일, 다단계 하위 파일, 같은 이름의 다른 경로 파일, 공백 또는 한글 경로, 읽기 실패 상태를 포함해야 한다.

### Key Entities

- **Worktree Session**: 사용자가 특정 worktree에서 agent 작업 결과와 파일을 검토하는 화면 단위이며, Files 섹션을 포함한다.
- **Files Tree Node**: Files 섹션의 파일트리에 표시되는 디렉터리 또는 파일 항목이며, 파일 항목은 선택 가능한 조회 대상이다.
- **Selected File**: 사용자가 파일트리에서 마지막으로 선택한 파일이며, 전체 경로와 표시 내용이 일치해야 한다.
- **File Viewer State**: 선택한 파일의 내용, 로딩 상태, 빈 상태, 오류 상태를 표현하는 Files 섹션의 표시 상태다.

## Constitution Alignment *(mandatory)*

- **Monorepo boundary**: 범위는 `apps/agentic-workbench`의 worktree session page Files 섹션이다. 다른 앱으로의 직접 import나 cross-app 공유 변경은 필요하지 않은 것으로 가정한다.
- **Frontend layering**: 화면 조립과 panel 배치는 AW의 `pages` 경계를 따르고, 파일 선택이라는 사용자 상호작용은 관련 `features` 경계에 둔다. 파일트리와 선택 상태에 필요한 domain model 또는 helper는 AW 내부 `entities` 또는 `shared` 경계를 따른다.
- **Backend boundary**: 파일 내용 조회 경로가 Tauri backend를 통과하는 경우 Tauri command는 경계 검증만 담당하고, 파일 조회와 root/path 안전성은 application service와 infrastructure adapter 경계를 통해 유지해야 한다.
- **Shared core vs UI**: 이 수정은 AW 한 화면의 회귀 복구가 주 목적이므로 신규 shared package는 도입하지 않는다. 파일 경로 정규화나 선택 상태 helper가 재사용 가치가 있을 경우에도 우선 AW 내부에서 검증한다.
- **Persistence and safety**: 파일 조회는 현재 worktree/session 소유 범위 안에서만 허용되어야 하며, 하위 경로 선택이 worktree root 밖의 파일을 조회하게 만들어서는 안 된다. 읽기 실패와 허용되지 않는 경로는 안전한 오류 상태로 표시한다.
- **Documentation and Storybook**: 재사용 가능한 파일트리 또는 file viewer 상태 컴포넌트가 변경되면 Storybook에 루트 파일, 하위 파일, 오류 상태를 등록하거나 갱신한다. 아키텍처 또는 사용자 흐름 문서가 필요하면 `docs/*.md`에 영어 파일명과 한국어 본문으로 작성한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 릴리즈 버전 검증에서 대표 fixture의 한 단계 하위 디렉터리 파일 100%가 파일트리 선택 후 올바른 내용으로 표시된다.
- **SC-002**: 릴리즈 버전 검증에서 대표 fixture의 두 단계 이상 중첩된 하위 파일 100%가 파일트리 선택 후 올바른 내용으로 표시된다.
- **SC-003**: 같은 파일명이 서로 다른 경로에 존재하는 fixture에서 선택한 경로와 다른 파일 내용이 표시되는 사례가 0건이다.
- **SC-004**: 공백 또는 한글이 포함된 하위 경로 fixture의 파일 100%가 올바르게 조회된다.
- **SC-005**: 루트 파일 조회, 하위 파일 조회, 다시 루트 파일 조회를 반복하는 사용자 흐름이 5회 연속 실패 없이 완료된다.
- **SC-006**: 읽을 수 없는 하위 파일을 선택했을 때 이전 파일 내용이 잘못 유지되는 사례가 0건이며, 사용자는 오류 상태를 확인할 수 있다.
- **SC-007**: 이 수정 후 worktree session page의 Files 섹션 외 주요 영역에 대한 기존 대표 검증에서 회귀가 0건이다.

## Assumptions

- "릴리즈 버전"은 사용자가 배포 또는 패키징된 앱 형태에서 경험하는 동작을 의미하며, 완료 검증은 개발 실행 환경만이 아니라 릴리즈에 준하는 실행 환경을 포함한다.
- 문제의 핵심은 파일트리에 하위 디렉터리 파일이 표시되더라도 선택 후 내용 조회가 실패하거나 잘못된 파일을 표시하는 동작이다.
- Files 섹션은 이미 루트 파일 조회 기능과 기본 파일트리 표시 기능을 제공하며, 이번 범위는 하위 디렉터리 파일 조회 회귀 수정에 집중한다.
- 새로운 파일 편집, 파일 생성, 삭제, 검색, 필터링 기능은 이번 범위에 포함하지 않는다.
- 파일 내용 표시 방식, 지원 파일 크기, binary 파일 처리 정책은 기존 Files 섹션의 정책을 따른다.
