# 조사: Workspace 패널 숨김 및 크기 저장

## 결정 1: 레이아웃 설정을 Agent Run 설정과 분리한다

- **결정**: `WorkspaceLayoutSettings` 전용 도메인 모델, repository port, 서비스, JSON 저장소와 Tauri 명령을 만든다.
- **근거**: 기존 `AgentRunSettings`는 에이전트·권한·Ralph Loop 설정을 Worktree 경로별로 저장한다. 레이아웃을 여기에 추가하면 서로 무관한 저장 갱신이 같은 객체를 덮어쓸 위험이 있고 책임 경계가 흐려진다.
- **검토한 대안**: `AgentRunSettings`에 레이아웃 필드를 추가한다. 기존 저장 경로를 재사용할 수 있지만 UI 레이아웃과 에이전트 실행 설정의 수명과 소유 책임이 다르므로 채택하지 않는다.

## 결정 2: B 폭만 픽셀 단위로 저장하고 A는 계산한다

- **결정**: 바깥 Workspace B 폭과 Git, Files, Markdown, Speckit 내부의 B 폭만 정수 픽셀 값으로 저장한다. 각 A는 현재 컨테이너에서 B와 핸들 폭을 제외한 나머지 공간을 사용한다.
- **근거**: `A:B = *:1` 요구를 직접 만족하고, 창 크기가 달라져도 A가 가용 공간을 흡수한다. 비율을 저장하면 창 크기 변화 후 B가 의도와 다르게 커지거나 작아진다.
- **검토한 대안**: 두 영역의 백분율 비율 저장. 구현은 단순하지만 B 고정 요구와 맞지 않는다. A와 B 폭을 모두 저장. 화면 크기 변경 때 충돌이 발생하므로 채택하지 않는다.

## 결정 3: 현재 화면에서는 제한하되 선호 폭을 덮어쓰지 않는다

- **결정**: 저장된 B 폭이 컨테이너 크기·최소 A/B 크기와 충돌하면 렌더링에 사용하는 폭만 제한한다. 줄어든 화면에서 제한된 값으로 저장 값을 다시 쓰지 않는다.
- **근거**: 작은 창을 한 번 열었다고 넓은 화면에서 쓰던 사용자의 B 선호 폭이 사라지지 않는다.
- **검토한 대안**: 제한한 값을 즉시 저장. 단순하지만 창 크기 변화가 사용자의 저장 설정을 파괴하므로 채택하지 않는다.

## 결정 4: 선택 없음 상태는 버튼 그룹으로 제공한다

- **결정**: Git, Files, Markdown, Speckit 제어 요소는 화면 오른쪽 세로 버튼 그룹으로 구현하고, 활성 버튼을 다시 누르면 선택이 해제되게 한다.
- **근거**: 전통적인 탭 목록은 항상 하나의 탭이 선택된다는 기대를 가진다. 선택 없음 상태를 올바르게 알리고 키보드 접근성을 유지하려면 토글 가능한 버튼 그룹이 적합하다.
- **검토한 대안**: 기존 `tablist`에 선택 없음 상태만 추가. `tabpanel`과 선택 관계가 불명확해지고 보조 기술 사용자에게 혼란을 주므로 채택하지 않는다.

## 결정 5: 저장은 레이아웃이 안정된 시점에 한다

- **결정**: 저장된 설정을 먼저 읽어 분할에 적용한 뒤, 사용자의 드래그가 끝났거나 레이아웃 변경이 안정된 시점에만 저장한다. 초기 기본 레이아웃이 아직 저장된 설정을 읽기 전에 저장되는 것을 막는다.
- **근거**: 초기 렌더의 기본 폭이 사용자가 설정한 폭을 덮어쓰지 않으며, 드래그 중 반복적인 파일 쓰기를 줄인다.
- **검토한 대안**: 모든 레이아웃 변경 이벤트에서 즉시 저장. 구현은 쉽지만 초기화 경합과 불필요한 저장이 생겨 채택하지 않는다.

## 현재 구현 근거

- 바깥 분할은 `apps/agentic-workbench/src/pages/project-worktree-session/ui/project-worktree-session-page.tsx`에 있으며 현재 40% 기본값만 사용하고 저장하지 않는다.
- Git, Files, Speckit, Markdown 내부 분할은 `apps/agentic-workbench/src/features/worktree-workspace/ui/worktree-workspace-panel.tsx`에 있으며 각각 기본 비율만 사용하고 저장하지 않는다.
- 기존 Worktree별 JSON 저장 패턴은 `domain/agent_run_settings*`, `application/agent_run_settings_service.rs`, `infrastructure/json_agent_run_settings_repository.rs`, `inbound/tauri_commands.rs`에 있다.
