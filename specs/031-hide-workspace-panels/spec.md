# Feature Specification: Hide Workspace Panels

**Feature Branch**: `[031-hide-workspace-panels]`

**Created**: 2026-07-24

**Status**: Draft

**Input**: User description: "git, files, markdown, speckit 을 표시하는 패널을 완전히 숨길 수 있는 기능을 제공하려고 합니다. 버튼들을 화면의 가장 오른쪽에 세로로 배치하고 배치할 때 90도 회전에서 배치합니다. 아무것도 선택하지 않을 수 있도록 하고 아무것도 선택하지 않았을 경우에는 화면에 표시되지 않도록 합니다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 보조 패널을 완전히 숨기기 (Priority: P1)

사용자는 Worktree Session의 작업 화면을 넓게 사용하기 위해 Git, Files, Markdown, Speckit 보조 패널 중 선택된 패널을 해제하여 모든 보조 패널을 화면에서 완전히 숨길 수 있다.

**Why this priority**: 보조 도구가 필요하지 않은 작업에서는 패널이 화면 공간을 차지하지 않아야 에이전트 작업 표면에 집중할 수 있다.

**Independent Test**: 네 패널 중 하나가 열려 있는 상태에서 해당 선택을 해제하고, 네 패널의 콘텐츠와 예약된 패널 영역이 모두 사라지는지 확인한다.

**Acceptance Scenarios**:

1. **Given** Git, Files, Markdown, Speckit 중 하나가 표시된 상태, **When** 사용자가 현재 선택된 패널의 제어 버튼을 다시 선택하면, **Then** 선택 상태가 해제되고 어떤 보조 패널도 표시되지 않는다.
2. **Given** 어떤 보조 패널도 선택되지 않은 상태, **When** 사용자가 작업 화면을 확인하면, **Then** Git, Files, Markdown, Speckit 콘텐츠와 빈 패널 컨테이너가 화면 공간을 차지하지 않는다.
3. **Given** 어떤 보조 패널도 선택되지 않은 상태, **When** 사용자가 네 제어 버튼 중 하나를 선택하면, **Then** 선택한 패널만 표시된다.

---

### User Story 2 - 화면 오른쪽에서 패널 선택하기 (Priority: P2)

사용자는 화면 가장 오른쪽에 세로로 정렬된 제어 버튼을 통해 Git, Files, Markdown, Speckit 패널을 빠르게 선택하거나 선택을 해제할 수 있다. 버튼의 읽을 수 있는 식별 표시는 90도 회전된 방향으로 배치된다.

**Why this priority**: 제어 영역을 화면 가장자리로 이동하면 보조 패널 선택 기능은 유지하면서 중앙 작업 영역의 가로 공간을 우선할 수 있다.

**Independent Test**: Worktree Session을 열어 네 패널 제어 버튼이 화면의 가장 오른쪽에 세로로 표시되는지, 각 버튼의 식별 표시가 90도 회전되어 있는지, 각 버튼으로 대상 패널을 전환할 수 있는지 확인한다.

**Acceptance Scenarios**:

1. **Given** Worktree Session 화면이 열린 상태, **When** 사용자가 보조 패널 제어 영역을 확인하면, **Then** Git, Files, Markdown, Speckit 제어 버튼은 화면 가장 오른쪽에서 세로로 정렬되어 있다.
2. **Given** 제어 버튼이 표시된 상태, **When** 사용자가 각 버튼의 식별 표시를 확인하면, **Then** 식별 표시는 90도 회전된 방향으로 배치되어 있고 어느 버튼이 어떤 패널을 여는지 구분할 수 있다.
3. **Given** 한 패널이 선택된 상태, **When** 사용자가 다른 패널의 제어 버튼을 선택하면, **Then** 이전 패널은 닫히고 새로 선택한 패널만 표시된다.

---

### User Story 3 - 패널 없는 상태에서도 기존 작업 유지하기 (Priority: P3)

사용자는 보조 패널을 모두 숨겨도 에이전트 실행, 프롬프트 입력, 작업 세션의 핵심 흐름을 계속 이용할 수 있다.

**Why this priority**: 보조 패널 숨김은 화면 구성의 변화여야 하며, 세션의 핵심 작업 흐름을 중단시키면 안 된다.

**Independent Test**: 보조 패널을 모두 숨긴 뒤 프롬프트 입력과 실행 상태 확인을 수행하고, 이후 패널 하나를 다시 열어 해당 패널의 기존 기능을 사용할 수 있는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 어떤 보조 패널도 선택되지 않은 상태, **When** 사용자가 에이전트 작업을 계속하면, **Then** 프롬프트 입력과 실행 상태 확인은 기존과 같이 가능하다.
2. **Given** 어떤 보조 패널도 선택되지 않은 상태, **When** 사용자가 Git, Files, Markdown, Speckit 중 하나를 다시 선택하면, **Then** 선택한 패널의 기존 표시 및 상호작용 기능을 사용할 수 있다.

---

### User Story 4 - Worktree별 패널 크기 유지하기 (Priority: P4)

사용자는 각 Worktree의 작업 성격에 맞게 작은 보조 패널 영역(B)의 크기와 Git, Files, Markdown, Speckit 각 패널 내부의 작은 영역(B) 크기를 조절하고, 같은 Worktree를 다시 열었을 때 마지막으로 조절한 B 크기로 작업을 이어갈 수 있다. 모든 짝의 큰 작업 영역(A)은 저장된 크기를 갖지 않고 항상 남은 공간을 채운다. 다른 Worktree의 B 크기는 현재 Worktree의 설정에 영향을 주지 않는다.

**Why this priority**: 코드 탐색, Markdown 검토, Git 확인에 필요한 보조 영역과 각 패널 내부 영역의 크기는 Worktree마다 다르므로, 사용자가 반복해서 같은 B 크기를 조절하지 않아야 한다. A가 남은 공간을 유연하게 사용하면 화면 크기 변화에도 핵심 작업 공간을 우선할 수 있다.

**Independent Test**: 한 Worktree에서 바깥 보조 패널과 Git, Files, Markdown, Speckit 내부의 조절 가능한 B 크기를 각각 조절한 뒤 세션을 닫고 다시 열어 모든 B 크기가 복원되고 각 A가 남은 공간을 채우는지 확인한다. 이어서 다른 Worktree에서 별도 B 크기를 정한 후 각 Worktree를 다시 열어 서로의 설정이 독립적인지 확인한다.

**Acceptance Scenarios**:

1. **Given** 보조 패널이 표시된 Worktree Session, **When** 사용자가 B 영역의 크기를 조절하면, **Then** B의 조절된 크기가 즉시 반영되고 A는 나머지 공간을 채운다.
2. **Given** 사용자가 한 Worktree에서 B 크기를 조절한 상태, **When** 해당 Worktree의 세션을 닫았다가 다시 열면, **Then** 마지막으로 조절한 B 크기가 복원되고 A는 남은 공간을 사용한다.
3. **Given** 서로 다른 두 Worktree에서 각각 다른 B 크기를 조절한 상태, **When** 각 Worktree를 다시 열면, **Then** 각 Worktree는 자신의 마지막 B 크기로 복원되고 다른 Worktree의 B 크기를 사용하지 않는다.
4. **Given** 보조 패널을 숨긴 상태에서 이전에 B 크기를 조절한 Worktree, **When** 사용자가 보조 패널을 다시 표시하면, **Then** 해당 Worktree에 저장된 마지막 B 크기로 표시되고 A는 나머지 공간을 채운다.
5. **Given** 사용자가 한 Worktree의 Git, Files, Markdown 또는 Speckit 패널 내부에서 B 크기를 조절한 상태, **When** 해당 Worktree를 다시 열고 같은 패널을 표시하면, **Then** 해당 패널 내부의 마지막 B 크기가 복원되고 내부 A는 남은 공간을 사용한다.
6. **Given** 사용자가 같은 Worktree에서 서로 다른 두 보조 패널의 내부 B 크기를 다르게 조절한 상태, **When** 각 패널을 다시 표시하면, **Then** 각 패널은 자신의 내부 B 크기를 독립적으로 복원한다.

### Edge Cases

- 이미 선택된 패널의 제어 버튼을 빠르게 반복 선택해도, 마지막 선택 결과와 일치하는 하나의 표시 상태 또는 완전한 숨김 상태만 남아야 한다.
- 패널을 모두 숨긴 상태에서 화면 크기가 변경되어도, 숨겨진 패널의 빈 여백이 다시 생기거나 제어 버튼이 화면 밖으로 밀려나면 안 된다.
- 패널별 데이터가 없거나 로딩 또는 오류 상태여도, 사용자는 해당 패널을 선택 해제하여 보조 패널 영역 전체를 숨길 수 있어야 한다.
- 패널을 숨기고 다시 열 때, 기존 패널이 제공하던 접근성 이름과 키보드로 조작 가능한 선택 흐름이 유지되어야 한다.
- 저장된 B 크기가 현재 화면에서 사용할 수 있는 공간보다 크거나 작아진 경우에도, B와 A는 모두 조작 가능한 크기로 표시되어야 하며 A는 남은 공간을 사용해야 한다.
- 특정 보조 패널에 내부 분할 영역이 없으면, 그 패널에는 내부 크기 설정을 만들거나 복원하려고 시도하지 않아야 한다.
- 처음 열거나 저장된 패널 크기를 찾을 수 없는 Worktree에서는, 사용자가 즉시 조절할 수 있는 안전한 기본 크기로 패널을 표시해야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 Worktree Session에서 Git, Files, Markdown, Speckit 패널 각각에 대응하는 제어 버튼을 제공해야 한다.
- **FR-002**: 시스템은 네 패널 제어 버튼을 화면의 가장 오른쪽에 세로로 배치해야 한다.
- **FR-003**: 시스템은 각 제어 버튼의 식별 표시를 90도 회전된 방향으로 배치해야 하며, 사용자가 대상 패널을 구분할 수 있어야 한다.
- **FR-004**: 시스템은 사용자가 네 패널 중 정확히 하나를 선택하거나 아무 패널도 선택하지 않을 수 있게 해야 한다.
- **FR-005**: 시스템은 현재 선택된 패널의 제어 버튼을 다시 선택하면 해당 선택을 해제해야 한다.
- **FR-006**: 시스템은 아무 패널도 선택되지 않은 경우 Git, Files, Markdown, Speckit 패널의 콘텐츠와 패널 컨테이너를 표시하지 않아야 하며, 그 영역의 화면 공간을 확보해야 한다.
- **FR-007**: 시스템은 사용자가 선택되지 않은 패널의 제어 버튼을 선택하면 해당 패널만 표시하고 다른 보조 패널은 숨겨야 한다.
- **FR-008**: 시스템은 패널 선택 또는 선택 해제가 에이전트 실행, 프롬프트 입력, 현재 Worktree Session의 핵심 작업 흐름을 중단시키지 않게 해야 한다.
- **FR-009**: 시스템은 패널을 다시 선택했을 때 해당 패널의 기존 데이터 표시, 빈 상태, 로딩 상태 및 오류 상태 동작을 유지해야 한다.
- **FR-010**: 시스템은 선택됨과 선택 해제됨 상태를 보조 기술과 키보드 사용자도 구분하고 조작할 수 있게 해야 한다.
- **FR-011**: 시스템은 사용자가 표시된 작은 보조 패널 영역(B)의 크기를 조절할 수 있게 해야 한다.
- **FR-012**: 시스템은 큰 작업 영역(A)과 작은 보조 패널 영역(B)을 `A:B = *:1` 관계로 배치해야 한다. B만 사용자가 조절한 크기를 기준으로 고정하고, A는 B를 제외한 나머지 사용 가능 공간을 채워야 한다.
- **FR-013**: 시스템은 사용자가 조절한 B 영역의 크기를 해당 Worktree의 식별 범위에 연결해 저장해야 한다.
- **FR-014**: 시스템은 사용자가 같은 Worktree의 세션을 다시 열면 마지막으로 저장된 B 영역 크기를 복원하고, A는 당시의 남은 공간을 사용하게 해야 한다.
- **FR-015**: 시스템은 각 Worktree의 저장된 B 크기를 다른 Worktree와 독립적으로 유지해야 한다.
- **FR-016**: 시스템은 보조 패널이 숨겨진 동안에도 해당 Worktree의 마지막 B 크기를 유지하고, 패널을 다시 표시하면 그 크기를 적용해야 한다.
- **FR-017**: 시스템은 저장된 B 크기를 현재 화면에서 안전하게 적용할 수 없는 경우, A와 B를 모두 사용할 수 있는 범위로 B를 조정하고 A가 남은 공간을 사용하게 해야 한다.
- **FR-018**: 시스템은 Git, Files, Markdown, Speckit 패널 내부에 사용자가 조절할 수 있는 분할 영역이 있는 경우, 각 내부 분할을 `A:B = *:1` 관계로 배치해야 한다. 해당 내부 B만 사용자가 조절한 크기를 기준으로 고정하고, 내부 A는 남은 공간을 채워야 한다.
- **FR-019**: 시스템은 각 보조 패널 내부에서 사용자가 조절한 B 크기를 Worktree와 패널 종류에 연결해 저장해야 한다.
- **FR-020**: 시스템은 같은 Worktree에서 각 보조 패널을 다시 표시하거나 세션을 다시 열면, 해당 패널 종류에 저장된 내부 B 크기를 복원해야 한다.
- **FR-021**: 시스템은 같은 Worktree 안에서도 Git, Files, Markdown, Speckit의 내부 B 크기를 서로 독립적으로 유지해야 하며, 다른 Worktree의 내부 B 크기를 적용하지 않아야 한다.
- **FR-022**: 시스템은 내부 B 크기를 현재 화면에서 안전하게 적용할 수 없는 경우, 내부 A와 B를 모두 사용할 수 있는 범위로 내부 B를 조정하고 내부 A가 남은 공간을 사용하게 해야 한다.
- **FR-023**: 시스템은 Worktree Session 창의 위치와 내부 크기를 Worktree별로 저장하고 같은 Worktree를 다시 열 때 복원해야 한다. 구현과 검증은 [창 상태 복원 시도 기록](../../docs/window-state-recovery-attempts.md)을 참고해 현재 복원 실패 원인을 해결한 뒤 완료로 간주해야 한다.

### Key Entities

- **보조 패널 선택 상태**: 현재 Worktree Session에서 표시할 보조 패널을 나타내는 상태. Git, Files, Markdown, Speckit 중 하나이거나 선택 없음일 수 있다.
- **패널 제어 버튼**: 특정 보조 패널을 표시하거나, 해당 패널이 이미 선택된 경우 선택을 해제하는 화면 가장자리의 제어 요소.
- **A 작업 영역**: 에이전트 작업과 핵심 세션 흐름을 위한 큰 영역. 독립 크기는 저장하지 않으며 B를 제외한 남은 공간을 사용한다.
- **B 보조 패널 영역**: Git, Files, Markdown, Speckit 중 선택된 패널을 표시하는 작은 영역. 사용자가 조절한 크기를 Worktree별로 저장하고 복원한다.
- **Worktree별 B 크기 설정**: 특정 Worktree에 연결된 마지막 B 영역 크기. 패널 선택 상태와 별도로 유지되며, 같은 Worktree의 다음 세션에서 복원된다.
- **패널 내부 B 크기 설정**: Git, Files, Markdown 또는 Speckit 패널 안의 조절 가능한 작은 영역(B)에 대한 마지막 크기. Worktree와 패널 종류에 함께 연결되며, 해당 패널 내부의 A와 B는 `*:1` 원칙을 따른다.

## Constitution Alignment *(mandatory)*

- **Monorepo boundary**: 범위는 `apps/agentic-workbench`의 Worktree Session 화면으로 한정하며, 다른 앱 또는 공용 패키지 변경은 필요하지 않다.
- **Frontend layering**: 화면 배치와 세션 구성은 `pages`에 유지하고, 패널 선택·해제 상호작용은 `features`에 두며, 재사용 가능한 제어 요소는 적절한 `shared` 또는 기존 UI 프리미티브를 사용한다.
- **Backend boundary**: Worktree별 크기 설정을 영속화해야 한다. 저장 또는 복원이 필요하면 inbound 경계는 현재 Worktree 범위를 검증하고, application은 설정 처리 규칙을 담당하며, infrastructure는 저장 세부 사항을 담당한다.
- **Shared core vs UI**: 다른 앱에서 같은 패널 선택 모델이 실제로 필요해지기 전까지는 앱 내부에 유지하며, 공용 UI 패키지를 새로 만들지 않는다.
- **Persistence and safety**: 숨김 또는 표시 전환은 현재 Worktree Session에만 적용한다. 저장하는 값은 바깥 B 크기와 패널 종류별 내부 B 크기뿐이며, 모두 현재 Worktree 식별 범위에만 연결한다. A의 크기는 저장하지 않고 현재 화면의 남은 공간에서 계산하며, 다른 Worktree·세션·파일·에이전트 소유 범위에 영향을 주지 않는다.
- **Documentation and Storybook**: 재사용 가능한 패널 제어 요소가 만들어지면 적절한 Atomic Design 분류의 Storybook 예제를 추가한다. 사용자 동작이 문서화된 기존 설계와 달라지면 한국어 프로젝트 문서를 갱신한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 검증한 모든 Worktree Session에서 사용자는 1회 선택 해제 동작으로 네 보조 패널을 모두 숨길 수 있다.
- **SC-002**: 보조 패널이 모두 숨겨진 검증 화면의 100%에서 숨겨진 패널 콘텐츠 또는 빈 패널 영역이 화면 공간을 차지하지 않는다.
- **SC-003**: 검증한 모든 화면 크기에서 네 패널 제어 버튼은 화면의 가장 오른쪽에 세로로 표시되고, 각 대상 패널을 구분할 수 있다.
- **SC-004**: 검증한 모든 패널 전환에서 사용자가 1회 선택 동작으로 원하는 Git, Files, Markdown 또는 Speckit 패널을 표시할 수 있다.
- **SC-005**: 보조 패널을 숨긴 상태로 수행한 대표 프롬프트 입력 및 실행 확인 흐름의 100%가 기존 핵심 작업 흐름을 중단하지 않고 완료된다.
- **SC-006**: 키보드와 보조 기술을 이용한 검증에서 사용자는 네 제어 버튼의 대상과 선택 상태를 100% 구분하고 선택 또는 선택 해제를 완료할 수 있다.
- **SC-007**: 검증한 모든 Worktree에서 사용자가 조절한 B 크기는 같은 Worktree를 다시 열었을 때 100% 복원되고, A는 남은 공간을 사용한다.
- **SC-008**: 서로 다른 B 크기를 지정한 대표 Worktree 10개를 다시 열었을 때, 100%가 자신에게 저장된 B 크기를 복원하며 다른 Worktree의 크기를 적용하지 않는다.
- **SC-009**: 저장된 B 크기가 현재 화면에 맞지 않는 검증 환경에서 100%의 경우 A와 B를 모두 조작할 수 있고, A는 B를 제외한 남은 공간을 사용한다.
- **SC-010**: 내부 분할을 제공하는 Git, Files, Markdown, Speckit 패널의 검증에서 사용자가 조절한 내부 B 크기는 같은 Worktree에서 해당 패널을 다시 표시하거나 세션을 다시 열었을 때 100% 복원되고, 내부 A는 남은 공간을 사용한다.
- **SC-011**: 한 Worktree에서 서로 다른 내부 B 크기를 지정한 Git, Files, Markdown, Speckit 패널을 검증했을 때, 100%가 자신에게 저장된 내부 B 크기만 복원한다.
- **SC-012**: 대표 Worktree 10개에서 창을 이동·리사이즈한 뒤 닫고 다시 열었을 때, 100%가 마지막 위치와 내부 크기로 복원된다. 화면 범위를 벗어나는 저장값은 접근 가능한 위치와 크기로 보정한다.

## Assumptions

- 대상 화면은 Agentic Workbench의 Worktree Session이며, "Git, Files, Markdown, Speckit"은 현재 제공되는 네 보조 패널을 의미한다.
- 보조 패널의 선택 상태는 현재 화면 수명 동안 유지하는 것으로 충분하지만, 사용자가 조절한 보조 패널 크기는 같은 Worktree를 다시 열어도 복원한다.
- 버튼의 90도 회전은 아이콘 자체를 임의로 회전하는 것이 아니라, 사용자가 읽는 패널 식별 표시의 방향을 뜻한다.
- 패널을 숨겨도 해당 패널의 데이터나 진행 중인 작업을 삭제하거나 취소하지 않는다.
- 새 기능은 Git, Files, Markdown, Speckit 패널 내부의 개별 기능과 데이터 로딩 규칙을 변경하지 않는다.
- 패널 크기는 네 보조 패널이 공유하는 B 영역의 크기를 뜻하며, 패널 종류마다 서로 다른 크기를 별도로 저장하는 것은 이번 범위에 포함하지 않는다.
- 패널 크기는 네 보조 패널이 공유하는 바깥 B 영역과, 각 보조 패널 내부에 존재하는 조절 가능한 B 영역을 뜻한다. 내부 B 크기는 패널 종류별로 별도 저장한다.
- 크기 유지 모델은 바깥 및 내부의 모든 조절 가능한 분할에서 `A:B = *:1`이다. B만 Worktree별로 고정·저장하고, A는 화면 크기와 B를 기준으로 남은 공간을 유동적으로 사용한다.
- 창 크기·위치 복원은 현재 구현에서 실제 동작이 검증되지 않았으며, `docs/window-state-recovery-attempts.md`에 기록된 재현·조사·재구현 절차를 완료해야 한다.
