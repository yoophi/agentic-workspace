# Feature Specification: 접기 및 크기 조절이 가능한 패널

**Feature Branch**: `028`

**Created**: 2026-07-11

**Status**: Draft

**Input**: User description: "collapsible and resizable panel 컴포넌트를 추가합니다. 사용자는 패널의 핸들을 이용하여 패널의 크기를 조절할 수 있습니다. 패널의 타이틀을 클릭하여 패널을 최소 상태로 접을 수 있으며 접었을 때도 최소 영역에 패널 정보(타이틀)가 표시되어 확인하고 펼 수 있습니다. 두 패널은 나란히 모두 펼쳐진 상태로 표시되며, 각 패널의 타이틀은 클릭 가능하고 두 패널 사이의 세로 구분선을 드래그하여 크기를 조절합니다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 세로 구분선으로 패널 크기 조절 (Priority: P1)

사용자는 나란히 펼쳐진 두 패널 사이의 세로 구분선을 끌어 필요한 콘텐츠가 보이도록 두 패널의 너비를 조절한다.

**Why this priority**: 제한된 화면 공간을 작업 맥락에 맞게 배분하는 것이 이 컴포넌트의 핵심 가치다.

**Independent Test**: 두 패널이 모두 펼쳐진 상태에서 세로 구분선을 좌우로 끌어 양쪽 패널의 너비가 전체 가용 너비 안에서 상호 보완적으로 변하는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 두 패널이 나란히 펼쳐져 있을 때, **When** 사용자가 세로 구분선을 오른쪽으로 끌면, **Then** 왼쪽 패널은 넓어지고 오른쪽 패널은 같은 가용 공간 안에서 좁아진다.
2. **Given** 두 패널이 나란히 펼쳐져 있을 때, **When** 사용자가 세로 구분선을 왼쪽으로 끌면, **Then** 오른쪽 패널은 넓어지고 왼쪽 패널은 같은 가용 공간 안에서 좁아진다.
3. **Given** 한쪽 패널이 허용된 최소 너비에 도달했을 때, **When** 사용자가 해당 패널을 더 좁히는 방향으로 구분선을 끌면, **Then** 구분선과 양쪽 패널은 허용 경계를 넘지 않는다.
4. **Given** 사용자가 크기 조절을 마쳤을 때, **When** 세로 구분선을 놓으면, **Then** 두 패널은 조절된 너비를 유지한다.

---

### User Story 2 - 제목을 눌러 패널 접기와 펼치기 (Priority: P1)

사용자는 각 패널의 클릭 가능한 제목 영역을 눌러 해당 패널을 최소 상태로 접고, 다시 눌러 이전 작업 크기로 펼친다.

**Why this priority**: 사용하지 않는 영역을 즉시 줄이면서도 패널의 존재와 용도를 잃지 않게 하는 핵심 상호작용이다.

**Independent Test**: 제목을 연속으로 눌러 콘텐츠가 숨겨진 최소 상태와 콘텐츠가 보이는 펼침 상태가 전환되고, 다시 펼쳤을 때 직전 크기가 복원되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 왼쪽 패널이 펼쳐져 있을 때, **When** 사용자가 제목 영역을 클릭하면, **Then** 콘텐츠는 숨겨지고 패널은 좁은 세로 최소 영역에 제목을 우측 끝으로 정렬한 뒤 −90도로 회전해 표시한다.
2. **Given** 패널이 최소 상태일 때, **When** 사용자가 회전된 제목 텍스트뿐 아니라 최소 상태 영역 안의 빈 공간을 포함한 어느 지점이든 활성화하면, **Then** 패널은 자신이 접히기 직전에 사용하던 너비로 펼쳐지고 콘텐츠가 다시 보인다.
3. **Given** 패널이 최소 상태일 때, **When** 사용자가 패널을 확인하면, **Then** 제목과 펼칠 수 있음을 나타내는 상태 정보를 식별할 수 있다.
4. **Given** 두 패널이 모두 펼쳐진 초기 상태일 때, **When** 사용자가 화면을 확인하면, **Then** 양쪽 패널의 제목과 그 사이의 세로 구분선을 식별할 수 있다.
5. **Given** 한 패널이 최소 상태일 때, **When** 사용자가 세로 구분선을 끌려고 하면, **Then** 패널 너비는 변경되지 않는다.
6. **Given** 오른쪽 패널이 펼쳐져 있을 때, **When** 사용자가 Panel B의 제목 영역을 클릭하면, **Then** Panel B는 오른쪽의 좁은 세로 최소 영역으로 접히고 제목은 우측 끝에 정렬된 상태로 −90도로 회전해 표시되며 Panel A는 나머지 가용 너비를 사용한다.
7. **Given** Panel B가 최소 상태일 때, **When** 사용자가 회전된 Panel B 제목을 클릭하면, **Then** Panel B는 접기 전 너비로 펼쳐지고 Panel A는 두 패널이 사용할 수 있는 나머지 너비로 돌아간다.
8. **Given** Panel A가 펼쳐져 있고 Panel B가 최소 상태일 때, **When** 사용자가 Panel A의 제목을 확인하거나 조작하려 하면, **Then** Panel A의 제목은 클릭할 수 없는 상태로 표시되고 Panel A는 계속 펼쳐져 있다.
9. **Given** Panel B가 펼쳐져 있고 Panel A가 최소 상태일 때, **When** 사용자가 Panel B의 제목을 확인하거나 조작하려 하면, **Then** Panel B의 제목은 클릭할 수 없는 상태로 표시되고 Panel B는 계속 펼쳐져 있다.
10. **Given** 사용자가 서로 다른 너비의 Panel A와 Panel B를 차례로 접었다 펼칠 때, **When** 각 패널을 다시 펼치면, **Then** 각 패널은 다른 패널의 저장 너비와 혼동되지 않고 자신이 마지막으로 접히기 직전에 사용하던 너비로 복원된다.

### User Story 3 - 다양한 콘텐츠와 조작 방식에서 사용 (Priority: P2)

사용자는 제목이나 콘텐츠 길이가 달라도 패널 정보를 확인하며 포인터와 키보드로 접기·펼치기를 수행한다.

**Why this priority**: 재사용 가능한 패널은 서로 다른 화면과 콘텐츠에서도 일관되고 접근 가능한 경험을 제공해야 한다.

**Independent Test**: 짧은 제목, 긴 제목, 빈 콘텐츠, 긴 콘텐츠를 각각 넣고 키보드만 사용해 접기와 펼치기를 완료할 수 있는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 제목이 최소 영역보다 길 때, **When** 패널이 최소 상태가 되면, **Then** 제목은 최소 영역을 침범하지 않으면서 패널을 식별할 수 있는 방식으로 표시된다.
2. **Given** 사용자가 키보드로 제목 영역에 초점을 이동했을 때, **When** 활성화 키를 누르면, **Then** 포인터 클릭과 동일하게 접힘 상태가 전환된다.
3. **Given** 패널 콘텐츠가 비어 있거나 매우 길 때, **When** 사용자가 패널을 조절하거나 접고 펼치면, **Then** 패널의 조작 영역과 제목은 계속 사용할 수 있다.

### Edge Cases

- 사용자가 허용된 최소 또는 최대 크기를 넘어 세로 구분선을 끌면 양쪽 패널은 해당 경계 크기에 머문다.
- 크기 조절 도중 포인터가 패널 또는 창 영역을 벗어나도 조작 종료 시 패널은 유효한 크기로 확정된다.
- 사용자가 패널을 접었다가 화면의 가용 공간이 줄어든 뒤 펼치면, 패널은 현재 공간의 최대 허용 크기를 넘지 않는 범위에서 복원된다.
- 제목이 비어 있으면 패널을 식별할 수 없으므로 패널 제공자는 비어 있지 않은 제목을 제공해야 한다.
- 빠르게 여러 번 제목을 활성화해도 최종 입력과 일치하는 하나의 안정된 접힘 상태가 표시된다.
- 최소 상태에서는 세로 구분선이 실수로 콘텐츠 영역을 다시 늘리지 않으며, 두 패널을 모두 펼쳐야 크기를 조절할 수 있다.
- 한 패널을 최소 상태로 접으면 다른 패널은 남은 가용 너비를 사용하고, 접힌 패널의 제목 영역은 계속 표시된다.
- 최소 상태 영역에서 제목 텍스트와 떨어진 빈 공간을 포인터로 활성화해도 제목 텍스트를 활성화한 것과 동일하게 패널이 펼쳐진다.
- 두 패널의 최소 너비 합계와 세로 구분선이 현재 가용 너비보다 큰 경우에도 제목과 상태 전환 조작은 접근 가능한 상태를 유지한다.
- 회전된 제목이 최소 영역의 높이보다 길면 제목은 영역 밖으로 넘치지 않으면서 패널을 식별할 수 있게 표시된다.
- 한 패널이 최소 상태인 동안 마지막으로 펼쳐진 패널의 제목을 포인터나 키보드로 활성화해도 두 패널이 모두 최소 상태가 되지 않는다.
- 패널이 접혀 있는 동안 전체 가용 너비가 줄어들어 저장된 너비를 그대로 복원할 수 없으면 현재 경계 안에서 가능한 가장 가까운 너비로 펼쳐진다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 각각 제목 영역과 콘텐츠 영역을 가진 두 패널을 좌우로 나란히 제공해야 한다.
- **FR-002**: 사용자는 두 패널 사이의 세로 구분선을 좌우로 끌어 양쪽 패널의 너비를 연속적으로 조절할 수 있어야 한다.
- **FR-003**: 시스템은 패널 크기가 패널 제공자가 정한 최소 크기보다 작거나 최대 크기보다 커지지 않도록 제한해야 한다.
- **FR-004**: 사용자는 다른 패널이 펼쳐져 있을 때 패널 제목 영역을 활성화하여 해당 패널의 펼침 상태와 최소 상태를 전환할 수 있어야 한다.
- **FR-005**: Panel A의 최소 상태는 왼쪽의 좁은 세로 영역만 유지하고 콘텐츠를 숨기며, 제목을 해당 영역의 우측 끝에 정렬하고 −90도로 회전해 표시해야 한다.
- **FR-005A**: Panel B의 최소 상태는 오른쪽의 좁은 세로 영역만 유지하고 콘텐츠를 숨기며, 제목을 해당 영역의 우측 끝에 정렬하고 −90도로 회전해 표시해야 한다.
- **FR-006**: 최소 상태의 패널 영역 전체는 제목 텍스트와 빈 공간을 구분하지 않고 사용자가 패널을 다시 펼칠 수 있는 하나의 조작 지점으로 유지되어야 한다.
- **FR-007**: 시스템은 Panel A와 Panel B 각각에 대해 접기 직전의 유효한 펼침 너비를 별도로 보존하고, 다시 펼칠 때 현재 가용 공간 안에서 해당 패널의 보존된 너비를 복원해야 한다.
- **FR-008**: 어느 한 패널이 최소 상태이면 세로 구분선을 통한 크기 조절을 비활성화하고, 두 패널이 모두 펼쳐진 후에만 너비 조절을 허용해야 한다.
- **FR-009**: 활성화 가능한 제목 영역은 포인터와 키보드 모두로 상태를 전환할 수 있어야 하며, 현재 접힘 상태와 조작 가능 여부가 보조 기술에 전달되어야 한다.
- **FR-010**: 세로 구분선은 시각적으로 식별 가능해야 하며, 좌우로 조절할 수 있음을 사용자가 인지할 수 있어야 한다.
- **FR-011**: 두 패널은 모두 펼쳐진 상태로 시작해야 하며, 화면별 용도에 맞는 초기 너비를 지정할 수 있어야 한다.
- **FR-012**: 패널은 빈 콘텐츠, 넘치는 콘텐츠, 긴 제목에서도 펼침 상태의 제목과 최소 상태의 회전된 제목 및 핵심 조작 영역을 사용할 수 있는 상태를 유지해야 한다.
- **FR-013**: 패널은 접힘 상태나 크기가 변경될 때 이를 사용하는 화면이 최신 상태를 파악할 수 있도록 변경 결과를 제공해야 한다.
- **FR-014**: 이 기능은 두 패널의 좌우 배치와 상호작용만 다루며, 세 개 이상 패널의 자동 배치, 크기 상태의 장기 저장, 창 간 상태 동기화는 포함하지 않아야 한다.
- **FR-015**: 한 패널이 최소 상태가 되면 다른 패널은 최소 상태 영역을 제외한 나머지 가용 너비를 사용해야 한다.
- **FR-016**: Panel A와 Panel B는 각각 독립적으로 최소화하고 다시 펼칠 수 있어야 하며, 최소 상태의 패널은 원래의 좌우 위치를 유지해야 한다.
- **FR-017**: 시스템은 항상 Panel A 또는 Panel B 중 최소 하나를 펼침 상태로 유지해야 한다.
- **FR-018**: 다른 패널이 최소 상태일 때 마지막으로 펼쳐진 패널의 제목 영역은 클릭과 키보드 활성화를 허용하지 않아야 하며, 비활성 상태를 시각적 표시와 보조 기술에 전달해야 한다.

### Key Entities

- **패널 상태**: 현재 펼침 또는 최소 여부, 현재 크기, 마지막 유효 펼침 크기, 허용 최소·최대 크기를 나타낸다.
- **패널 정보**: 사용자가 패널을 식별하는 비어 있지 않은 제목과 현재 상태를 나타내는 정보다.
- **크기 조절 동작**: 조절 시작 크기, 사용자의 이동량, 조절 방향, 경계 적용 후 결과 크기를 나타낸다.
- **패널 쌍**: 좌우 패널, 두 패널 사이의 세로 구분선, 전체 가용 너비와 양쪽 너비의 관계를 나타낸다.

## Constitution Alignment *(mandatory)*

- **Monorepo boundary**: 여러 앱에서 사용할 수 있는 공통 UI 범위가 대상이며, 앱별 화면 구성이나 앱 셸 동작은 범위에서 제외한다.
- **Frontend layering**: 공통 UI 기본 요소 영역이 패널 자체를 소유하고, 각 앱의 화면 영역은 제목·콘텐츠·크기 제약을 조합해 사용한다.
- **Backend boundary**: 이 기능은 화면 상호작용만 포함하며 백엔드 도메인, 명령, 파일 시스템 작업은 포함하지 않는다.
- **Shared core vs UI**: 특정 앱 상태에 의존하지 않는 패널 상태 규칙과 UI 계약만 공유하며, 앱별 콘텐츠와 배치는 공유 대상이 아니다.
- **Persistence and safety**: 크기 또는 접힘 상태의 영구 저장과 세션 동기화는 이번 범위에 포함하지 않는다.
- **Documentation and Storybook**: 재사용 컴포넌트로서 두 패널 펼침, 왼쪽 최소, 오른쪽 최소, 크기 경계, 회전된 긴 제목, 긴 콘텐츠, 빈 콘텐츠 상태를 확인할 수 있는 Storybook 사례가 필요하다. 별도 사용자 문서는 이번 범위에 포함하지 않는다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 대표 사용성 평가에서 사용자의 95% 이상이 안내 없이 10초 안에 세로 구분선을 찾아 두 패널의 너비를 조절한다.
- **SC-002**: 대표 사용성 평가에서 사용자의 95% 이상이 제목을 이용해 5초 안에 패널을 접고 다시 펼친다.
- **SC-003**: 왼쪽 또는 오른쪽 최소 상태를 확인한 사용자의 100%가 회전된 패널 제목과 접힘 상태를 식별하고 해당 패널을 다시 펼칠 수 있다.
- **SC-004**: 최소·최대 경계를 넘는 조작, 빠른 상태 전환, 가용 공간 변경을 포함한 정의된 경계 시나리오의 100%에서 패널이 유효하고 조작 가능한 상태를 유지한다.
- **SC-005**: 포인터 없이도 사용자의 100%가 제목 영역에 접근해 접기와 펼치기를 완료할 수 있다.
- **SC-006**: 두 패널 펼침, 왼쪽 최소, 오른쪽 최소, 최소·최대 너비, 긴 제목, 긴 콘텐츠, 빈 콘텐츠의 7가지 대표 상태가 독립적으로 확인 가능하다.
- **SC-007**: 한 패널이 최소 상태인 모든 검증 사례에서 세로 구분선 드래그로 양쪽 패널의 너비가 변경되지 않는다.
- **SC-008**: 포인터, 키보드, 빠른 반복 입력을 포함한 모든 상태 전환 검증 사례에서 최소 한 패널이 펼쳐진 상태로 유지된다.
- **SC-009**: 가용 공간이 유지되는 모든 접기·펼치기 검증 사례에서 각 패널은 접기 직전 너비와 동일한 너비로 복원된다.
- **SC-010**: 최소 상태 영역의 상단, 중앙, 하단을 각각 활성화하는 모든 검증 사례에서 패널이 동일하게 펼쳐진다.
- **SC-011**: 왼쪽과 오른쪽 패널의 모든 최소 상태 검증 사례에서 회전된 제목이 최소 상태 영역의 우측 끝에 정렬되어 표시된다.

## Assumptions

- 이 컴포넌트는 두 패널을 좌우로 배치하고 그 사이의 세로 구분선을 따라 너비를 조절하는 경우를 대상으로 한다.
- 펼침 상태에서는 제목 영역 전체를, 최소 상태에서는 좁은 패널 영역 전체를 접기·펼치기 조작 지점으로 사용하되, 해당 영역에 별도의 대화형 요소를 배치하는 경우 그 요소의 조작은 상태 전환을 일으키지 않는다.
- 최소 상태의 너비는 −90도로 회전한 제목과 상태 표시를 확인하고 활성화할 수 있는 정도로 패널 제공자가 정하며, Panel A와 Panel B 제목은 각각 최소 영역의 우측 끝에 정렬한다.
- 각 패널의 접기 전 너비는 컴포넌트가 화면에 유지되는 동안 패널별로 보존되며, 앱 재시작 이후의 영구 복원은 포함하지 않는다.
- 한 패널이 최소 상태가 되면 다른 패널이 남은 너비를 사용하며, 다시 펼치면 접기 전 너비 비율을 현재 가용 공간 안에서 복원한다.
- 마지막으로 펼쳐진 패널의 제목은 패널 식별 정보로 계속 표시되지만, 다른 패널을 먼저 펼치기 전까지 접기 조작은 제공하지 않는다.
